## Debt Categories

**Documentation Debt:**
- Missing API docs
- Missing examples
- Outdated diagrams
- Missing configuration docs

**Code Debt:**
- TODOs in code
- Hardcoded values that should be configurable
- Missing error handling
- Performance optimizations deferred

**Testing Debt:**
- Missing unit tests
- Missing integration tests
- Manual testing not automated

**Architecture Debt:**
- Temporary workarounds
- Violations of architecture patterns
- Missing abstractions
